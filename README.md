## 需求

1. 一个需要刷的编程猫账号
2. 至少110个EDU账号的Token
3. 脑子

## 关于EDU账号

不教怎么注册，可以使用[CET项目](https://github.com/Wangs-official/CodemaoEDUTools)来进行批量登录

## 使用提示

1. 由于需要点赞分，你的消息红点可能一直会保持在99+状态，请自行寻找办法清除
2. 此项目更适用于当前等级为LV1（积木小白）或者是LV2（潜力新星）的用户使用，并且所有分数均为0分，具体原因往下看
3. 使用此项目后果自负，本人不负责
4. 可能会对主页数据产生影响
5. 数据疑似不是实时统计的，应该是24小时更新一次

## 使用方法

安装 `requests` 库

在命令行中执行 `python main.py`

注意！由于数据统计延迟，程序会保存生成的25个作品ID，请在统计完成后执行`python delete.py`来删除他们

## 算法

编程猫官方算法为：

| 等级 | 称号 |   被点赞分    |    被收藏数     |  被再创作数  |
|:----:|:----:|:---------:|:-----------:|:-------:|
| Lv2 | 潜力新星 |      单作品点赞数超过10次      | |         |
| Lv3 | 进阶高手 |  ≥ 100分   |   ≥ 100分    | ≥ 100分  |
| Lv4 | 编程大佬 | ≥ 400分  |   ≥ 500分    | ≥ 500分  |
| Lv5 | 源码传说 | ≥ 1000分 |   ≥ 1000分   | ≥ 1000分 |

### 点赞分计算规则
每个作品单独记分，所有作品分数累加为当前得分

- 单个作品超过50赞：+20分
- 单个作品超过100赞：+40分

### 收藏分计算规则
累计被收藏达到一定数值，获得对应的分数

- 200～499 ——100分
- 500～1999 —— 500分
- 2000以上 —— 1000分

### 再创作分计算规则
累计被再创作达到一定数值，获得对应的分数

- 50～99 —— 100分
- 100～499 —— 500分
- 500以上—— 1000分

### 程序

其实最棘手的就是点赞分，一个作品最多加40分，为了达到1000分的门槛，只能创建25个作品，每个作品点满100个赞，最终获得分数为1000分（25X40=1000=1000）

剩下就很简单，累计收藏只需要达到2000以上就能直接加满1000分，那么收藏分也拿到了

再创作分，不多说

至于为什么说推荐那种全0分的来用，因为有分的话会溢出，我也不想写判断了
